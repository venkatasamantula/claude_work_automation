# CLAUDE Project Overview


## Instructions
Read /.claude folder with all the instruction before start of research.


## Purpose
This project is designed to support day-to-day activities through a set of purpose-built agents and reusable prompts. It is organized around the idea that many routine tasks can be accelerated by specialized assistants that each bring the right focus, voice, and skill set.

## What this project includes
- Daily activity workflows and examples.
- Agent definitions for specific tasks, with clear job duties and behavior guidelines.
- Prompt templates for repeatable, high-value actions.

## Core idea
The project is not a single application; it is a workspace for building and documenting agents that can help with routine work. Each agent is designed to handle a narrow scope efficiently, while the prompt templates help standardize how you invoke those agents.

## Agents and skills
Each agent in this project is defined to solve a particular problem or support a specific workflow. For example:
- A thought-leader research assistant that takes a short idea and turns it into a polished, evidence-backed paragraph.
- A reusable prompt template that generates concise social posts in a specific style.

Agents are tied to skills and rules such as:
- how to interpret user input,
- what tools and sources to use,
- output format and tone,
- and whether citations are required.

## Daily activities
Typical day-to-day work in this project includes:
- capturing quick ideas or thought starters,
- converting those ideas into shareable posts,
- researching supporting evidence,
- refining messaging,
- and building new agent templates for recurring tasks.

## How to use this project
1. Pick a task or workflow you want to automate.
2. Create or update an agent definition that describes the job clearly.
3. Add prompt templates for consistent invocation.
4. Test the output and refine the agent’s style, format, and source guidance.

## Why it matters
This project is meant to make everyday creative and knowledge work more repeatable and reliable, while still preserving the voice and style you want to express. By documenting agents and prompts, you can scale what works and avoid recreating the same patterns.

## Notes
- The names, formats, and exact implementation details can evolve as the project grows.
- Keep agent definitions simple, focused, and easy to update.
- Treat each agent as a small, reusable skill that can support many different daily tasks.
